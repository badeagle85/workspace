# Delivery-Note PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 1.1 프로젝트명
**Delivery-Note** - 거래명세서 자동 변환 시스템

### 1.2 목적
거래명세서 이미지를 업로드하면 OCR을 통해 텍스트를 추출하고, 제조사별로 상이한 품명을 통일된 ERP 품명으로 자동 변환하여 재고 관리 효율성을 높이는 시스템

### 1.3 문제 정의
- 동일한 제품이라도 제조사마다 품명 표기 방식이 다름
- 수작업으로 품명을 통일하는 데 많은 시간 소요
- 재고 관리 시 품명 불일치로 인한 혼란 발생

**예시:**
| 거래명세서 품명 | 변환된 ERP 품명 |
|----------------|----------------|
| CBR 100AF 2P (SEC-102 60A) | 누전차단기 2P100AF[小] 60A |
| ELB 2P 100AF 60A (LS) | 누전차단기 2P100AF[小] 60A |
| ELCB 100AF/60A 2P (현대) | 누전차단기 2P100AF[小] 60A |

## 2. 타겟 사용자

### 2.1 주요 사용자
- 전기자재 유통업체 직원
- 재고 관리 담당자
- ERP 시스템 입력 담당자

### 2.2 사용자 니즈
- 거래명세서 수기 입력 시간 절감
- 품명 통일로 정확한 재고 파악
- 실수로 인한 재입력 방지

## 3. 핵심 기능

### 3.1 이미지 업로드 및 OCR
- 거래명세서 이미지 업로드 (JPG, PNG, PDF)
- 다중 이미지 업로드 지원
- OCR을 통한 텍스트 추출
- 드래그 앤 드롭 지원

### 3.2 품명 자동 변환
- 제조사별 품명 → 통일된 ERP 품명 변환
- AI 기반 품명 매칭 (유사도 분석)
- 변환 규칙 데이터베이스 관리
- 수동 매칭 기능 (신규 품명 등록)

### 3.3 데이터 관리
- 변환된 데이터 저장 및 조회
- 거래처별 거래 이력 관리
- Excel/CSV 내보내기
- 검색 및 필터링

### 3.4 품명 매핑 관리
- 품명 변환 규칙 등록/수정/삭제
- 제조사별 품명 패턴 관리
- 표준 ERP 품명 목록 관리

## 5. 비기능 요구사항

### 5.1 성능
- 이미지 업로드: 10MB 이하 파일 3초 이내 업로드
- OCR 처리: 단일 이미지 5초 이내 처리
- 품명 변환: 100개 품목 2초 이내 변환

### 5.2 사용성
- 반응형 디자인 (데스크톱, 태블릿, 모바일)
- 직관적인 UI/UX
- 한국어 지원

### 5.3 확장성
- 새로운 제조사 품명 패턴 추가 용이
- API 기반 설계로 외부 시스템 연동 가능

## 6. 기술 스택

### 6.1 프론트엔드
- **Framework**: Next.js 14 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **State Management**: Zustand
- **Data Fetching**: TanStack Query (React Query)
- **UI Components**: shadcn/ui

### 6.2 백엔드
- **Runtime**: Next.js API Routes
- **Database**: PostgreSQL (Supabase 또는 자체 호스팅)
- **ORM**: Prisma
- **OCR**: Google Cloud Vision API / Tesseract.js
- **AI 매칭**: OpenAI API (품명 유사도 분석)

### 6.3 인프라
- **Hosting**: Vercel
- **Database**: Supabase / PlanetScale
- **Storage**: Cloudflare R2 / AWS S3
